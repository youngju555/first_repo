working directory 실제 작업 중인 파일들이 위치하는 영역
staging area 워킹 디렉토리에서 변경된 파일 중 다음 버전에 포함시킬 파일들을 선택적으로 추가하거나 제외할 수 있는 중간 준비 영역
repository 버전 이력과 파일들이 영구적으로 저장되는 영역. 모든 버전과 변경 이력이 기록됨. >> 모든 변화를 지켜봄

버전 >> commit (기여하다. 기록하다)




git init >> 저장소 초기화

git status >> 깃 상태 확인

git add . >> 워킹디렉토리에서 스테이징 에어리아로 넘기기. (모든 것) ( 권장되진 않음)

git commit -m "메세지 쓰셈"


git log 그동안의 기록들 보기

git은 로컬 저장소 내 모든 파일의 변경사항을 감시하고 있다.

로컬 (내가 접속하고 있는 기기 또는 시스템)  >> 서버로 리모트

git init 주의사항

git 저장소 안에 또다른 git 저장소를 만들지 말것 ( 다 꼬여버림) 

git config >> 보고서에 작성자 이름을 쓰는것. 누가 작성했는지. 깃허브에 sign in을 따로 해야함.
자리를 바꿀 때마다 이걸 다 제거를 해줘야 함.
